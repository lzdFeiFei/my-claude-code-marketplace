# Technical Terminology Reference

This document provides standardized Chinese translations for common technical terms to ensure consistency across translations.

## Translation Strategies

### Strategy 1: Keep Original (Most Common)
Use for widely recognized English terms in Chinese tech community.

### Strategy 2: Translate + Parentheses (First Mention)
Use for important concepts: Chinese Term (English Original)

### Strategy 3: Pure Chinese Translation
Use for common concepts with well-established Chinese terms.

## Usage Guidelines

### When Translating Technical Content

1. **First Mention Rule**
   - For important concepts, use: 中文翻译 (English Original)
   - Example: "状态管理 (State Management)"
   - Subsequent mentions: use Chinese only

2. **Code Context Rule**
   - In code blocks: keep all English
   - In explanations: use appropriate Chinese

3. **API/Function Names**
   - Always keep original: useState, fetch, Promise
   - In explanations, add context: "使用 `useState` Hook"

4. **Framework/Library Names**
   - Always keep original: React, Vue, Next.js

5. **Established Terms**
   - Use pure Chinese for well-known terms: 组件, 接口, 数据库

6. **Mixed Usage**
   - Some terms commonly used in both forms
   - "type" and "类型" often used interchangeably
   - Choose based on context and readability

### Consistency Tips

- Within a single document, stick to one translation for each term
- For series of documents, maintain consistency across all files
- When in doubt, prefer the more commonly used form in Chinese tech community
- Check official Chinese documentation of frameworks/tools for standard terms
